Writeup#

HTTP port 80#

visiting the home page of http://greenhorn.htb/ we are greeted with a welcome screen for aspiring web developers. the message is not very important, but at the bottom of the home page we find a link to the admin login panel.

  • http://greenhorn.htb/login.php

before we move any further, we start a gobuster scan in attempt to discover any hidden paths or files we would like to examine. the gobuster scan does not reveal any interesting paths.

navigataing to login.php reveals the server is running pluck 4.7.18. a quick google search reveals a public rce exploit. unfortunately it is an authenticated rce vulnerability, since we do not have credentials we will move passed this for now.

HTTP port 3000#

the service hosted on port 3000 is gitea, which convieniently requires no permissions and allows us to read the source code of the website hosted on port 80. looking at the commit history the author is “junior” with the message “First release of our source code”. there are surely some bugs or hardcoded credentials/tokens here.

after a minute or so of combing the file we come across

User flag#

following the pluck 4.7.18 exploit aforementioned, we can create a malicious .php file, in my case i used a php reverse shell file from pentestmonkey

archive the php file into a zip:

7z a rev.zip rev.php

navigate to http://greenhorn.htb/admin.php?action=installmodule and install the module. ensure that before you install the module you have your listener running, once the module is installed the reverse shell payload will be executed and you will have your shell!

although we have a shell, the user flag is owned by user junior which we currently do not have persmissions for.

tip: always try the path of least resistance first.. password reuse is common. in this case, we can easily priv esc to junior by reusing the previously found password iloveyou1

Root flag#

inside of juniors home directory we find a .pdf file labeled Using OpenVAS.pdf, using curl and python3 upload server we can easily exfiltrate this file for review on our attacking machine

reviewing the pdf

as user junior we cannot run any sudo commands, also, it appears that openvas is not installed on the system.

after enumerating the machine we determine that the likely path forward is somehow extracting the password from the pdf.

using tools from poppler-utils we simply try to extract the text from the pdf, which yields no fruitful results. the pixelated password is stored as an image, which we can extract using

pdfimages -all Using\ OpenVAS.pdf 0

with the image extracted, we begin our search on depixelating the image. searching for “password depixilator” on google we find the following repo https://github.com/spipm/Depix.

following the example usage:

python3 depix.py \
    -p images/testimages/testimage3_pixels.png \
    -s images/searchimages/debruinseq_notepad_Windows10_closeAndSpaced.png

we get a cleartext password!

after a few attempts we finally find that the password is all the words in a single string with no hyphens, no underscores, and no spaces.. not my favorite solution.
